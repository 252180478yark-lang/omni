# PRD — 投放复盘系统（Ad Review System）

> 模块代号：SP8-AdReview  
> 版本：v1.0  
> 日期：2026-04-07  
> 状态：草案

---

## 1. 背景与目标

### 1.1 业务痛点

当前投放工作流中，用户在巨量千川完成素材投放后，复盘环节完全依赖"脑子里回忆"：

- 投放数据（CSV导出）、视频分析结果、人群画像、知识库各自独立，无法交叉关联
- 优化经验无法沉淀，同样的错误可能反复犯
- 缺乏历史数据对比，无法追踪"优化前 vs 优化后"的效果变化
- 制作新视频时没有基于数据的指导，只能凭感觉

### 1.2 产品目标

构建一个**投放复盘闭环系统**，实现：

1. **数据整合**：将千川投放数据、视频分析结果、人群画像、圈包手法统一关联
2. **智能复盘**：AI 结合知识库 + 投放数据 + 视频分析自动生成优化建议
3. **经验沉淀**：复盘日志自动入知识库，形成可检索的投放经验库
4. **迭代追踪**：支持标记素材的优化版本，追踪优化前后效果变化
5. **指导创作**：在制作新视频时，可通过 RAG 检索历史投放经验获得指导

### 1.3 核心用户

内容投放操盘手（个人使用，自己查看日志和复盘结果）

## 2. 核心概念与实体关系

### 2.1 实体定义


| 实体                         | 说明           | 举例                                 |
| -------------------------- | ------------ | ---------------------------------- |
| **产品 (Product)**           | 投放推广的具体商品    | XXX面霜                              |
| **投放批次 (Campaign)**        | 一次完整的投放周期    | 2026-04-01 ~ 2026-04-03, XXX面霜春季推广 |
| **人群包 (AudiencePack)**     | 巨量云图圈出的DMP人群 | 25-35岁女性/护肤兴趣/竞品粉丝                 |
| **圈包手法 (TargetingMethod)** | 人群包的定向策略描述   | LLM生成的定向建议 + 人群画像Excel             |
| **素材 (Material)**          | 投放的视频素材      | 素材A-开箱.mp4                         |
| **投放数据 (AdMetrics)**       | 千川导出的CSV数据   | 消耗、点击率、转化率等                        |
| **视频分析 (VideoAnalysis)**   | SP6模块的分析结果   | 钩子力8分、内容价值7分                       |
| **复盘日志 (ReviewLog)**       | AI生成的复盘报告    | 包含数据总结、优化建议、经验标签                   |


### 2.2 实体关系

```
Product (产品)
  └─── Campaign (投放批次) [1:N — 一个产品多次投放]
         ├─── AudiencePack (人群包) [1:N — 一个批次多个人群包]
         │      ├─── TargetingMethod (圈包手法) [1:1]
         │      └─── Material (素材) [1:N — 一个人群包多条素材]
         │             ├─── AdMetrics (投放数据) [1:1 — CSV导入]
         │             ├─── VideoAnalysis (视频分析) [1:1 — 手动关联SP6]
         │             └─── parent_material_id [自引用 — 迭代追踪]
         └─── ReviewLog (复盘日志) [1:1 — AI生成]
```

## 3. 功能模块

### 3.1 模块一：投放批次管理

**描述**：创建和管理投放批次，作为复盘的基本单元。

**用户故事**：

- 作为用户，我要创建一个投放批次，填写产品名称、投放周期、总预算
- 作为用户，我要查看所有历史投放批次列表，按时间倒序
- 作为用户，我要能编辑和删除投放批次

**功能要点**：

- 创建批次时可选择已有产品或新建产品
- 批次状态流转：`草稿` → `数据已上传` → `已复盘` → `已归档`
- 支持按产品、时间范围、状态筛选

---

### 3.2 模块二：人群包与圈包手法

**描述**：在投放批次下管理人群包信息和圈包策略。

**用户故事**：

- 作为用户，我要为一个批次添加多个人群包，描述目标人群特征
- 作为用户，我要上传圈包手法（LLM建议文本 + 人群画像Excel）
- 作为用户，我要编辑人群包的描述信息

**功能要点**：

- 人群包字段：名称、描述（自由文本）、标签（年龄/性别/兴趣等结构化标签）
- 圈包手法：支持上传 LLM 生成的文本描述 + Excel 文件
- Excel 文件存储原文件，同时解析关键字段展示

---

### 3.3 模块三：素材与数据导入

**描述**：上传千川CSV数据，关联视频分析结果。

**用户故事**：

- 作为用户，我要上传千川导出的CSV文件，系统自动解析为素材列表
- 作为用户，我要为每条素材手动选择关联的视频分析结果（来自SP6）
- 作为用户，我要标记某条素材是上一条素材的"优化版本"

**CSV字段映射**（一行一条素材）：


| CSV列名     | 系统字段              | 类型      |
| --------- | ----------------- | ------- |
| 素材名称/素材ID | material_name     | string  |
| 消耗        | cost              | decimal |
| 展示次数      | impressions       | integer |
| 点击次数      | clicks            | integer |
| 前展        | front_impressions | integer |
| 点击率       | ctr               | decimal |
| 7日分享次数    | shares_7d         | integer |
| 评论次数      | comments          | integer |
| 播放数       | plays             | integer |
| 3秒播放数     | play_3s           | integer |
| 25%进度播放数  | play_25pct        | integer |
| 50%进度播放数  | play_50pct        | integer |
| 75%进度播放数  | play_75pct        | integer |
| 完播率       | completion_rate   | decimal |
| 新增A3      | new_a3            | integer |
| 消耗成本      | cost_per_result   | decimal |
| 新增A3占比    | a3_ratio          | decimal |


**功能要点**：

- CSV上传后自动解析列名（支持中文列名模糊匹配）
- 素材自动分配到当前人群包下
- 视频分析关联：展示SP6中已分析的视频列表，手动点选关联
- 迭代标记：选择"此素材是XX的优化版"，建立 `parent_material_id` 关系
- 支持计算衍生指标：3秒完播率 = play_3s / plays、互动率 = (shares_7d + comments) / plays

---

### 3.4 模块四：智能复盘引擎

**描述**：基于所有关联数据，AI 自动生成复盘日志和优化建议。

**用户故事**：

- 作为用户，当我上传完所有数据后，点击"生成复盘"，系统自动产出复盘日志
- 作为用户，我要看到 AI 结合知识库给出的优化建议
- 作为用户，我要在生成后可以编辑日志内容，然后保存

**复盘日志结构**：

```markdown
## 投放复盘 | {产品名} | {日期范围}

### 一、基础信息
- 产品：{product_name}
- 投放周期：{start_date} ~ {end_date}
- 总消耗：￥{total_cost}
- 人群包数量：{audience_count}
- 素材数量：{material_count}

### 二、人群包概览
#### 人群包A：{audience_name}
- 定向描述：{description}
- 圈包手法：{targeting_summary}

### 三、素材表现排名
| 排名 | 素材 | 消耗 | 展示 | 点击率 | 完播率 | 3秒播率 | 转化(A3) | CPM | CPC | GPM |
|------|------|------|------|--------|--------|---------|----------|-----|-----|-----|
| 1 | ... | ... | ... | ... | ... | ... | ... | ... | ... | ... |

### 四、视频分析关联
- 素材B（点击率最高5.8%）：钩子力 8/10，前3秒使用反问句式
- 素材C（点击率最低2.1%）：钩子力 4/10，开头无钩子

### 五、迭代追踪
- 素材A-v2 vs 素材A-v1：点击率 2.1% → 5.3%（+152%），变化：优化了前3秒钩子

### 六、AI优化建议
> 以下建议由 AI 结合投放数据、视频分析结果、历史复盘经验生成

1. **钩子策略**：...
2. **人群匹配**：...
3. **素材方向**：...
4. **避坑提醒**：...

### 七、经验标签
#产品类型 #人群特征 #有效策略 #无效策略
```

**AI生成逻辑**：

系统在生成优化建议时，构造如下 Prompt 发送给 LLM：

```
你是一位专业的巨量千川投放优化师。请基于以下数据生成投放复盘和优化建议。

## 投放数据
{各素材的投放指标数据}

## 视频分析结果
{关联的SP6视频分析报告：维度评分、钩子分析、画面分析等}

## 人群画像
{人群包描述和圈包手法}

## 历史经验（来自知识库RAG检索）
{根据当前产品/人群/素材类型，从复盘知识库中检索的相关历史经验}

请输出：
1. 数据亮点与问题诊断（哪些素材表现好/差，为什么）
2. 视频内容与投放数据的关联分析（什么内容特征带来高点击/高转化）
3. 人群匹配度评估（当前人群包与素材内容是否匹配）
4. 具体优化建议（下一轮素材制作方向、人群调整建议、投放策略建议）
5. 避坑清单（本次验证的无效做法）
6. 经验标签（用于知识库标记）
```

**关键特性**：

- RAG 检索范围：指定"投放复盘"知识库，检索相似产品/人群/素材类型的历史经验
- 迭代对比：如果存在标记了迭代关系的素材，自动计算前后指标变化并纳入分析
- 流式输出：生成过程实时展示
- 可编辑：生成后用户可在 Markdown 编辑器中修改

---

### 3.5 模块五：知识库沉淀

**描述**：复盘日志保存后自动写入知识库，供未来检索。

**用户故事**：

- 作为用户，保存复盘日志后，系统自动将其存入"投放复盘"知识库
- 作为用户，在制作新视频时，我可以在知识库问答中搜索历史复盘经验
- 作为用户，我要能检索"上次投护肤品什么钩子效果好"这类问题

**功能要点**：

- 自动创建/使用名为"投放复盘经验库"的专属知识库
- 每条复盘日志作为一个 document 入库
- 入库时增强 metadata：产品名、人群标签、经验标签、投放日期、关键指标
- metadata 增强检索：RAG 查询时可通过 metadata 过滤（如按产品、人群类型）
- 日志更新时同步更新知识库中的对应文档

**典型检索场景**：

- "上次投护肤品给25-35女性，什么钩子点击率最高？"
- "对比类视频在哪些人群表现好？"
- "上次视频犯了什么错误？"
- "什么样的画面和场景对转化率提升最好？"
- "给我这个产品的视频制作建议"

---

### 3.6 模块六：跨日志分析

**描述**：跨多次投放批次进行趋势分析和规律发现。

**用户故事**：

- 作为用户，我要看到同一个产品多次投放的指标趋势
- 作为用户，我要看到素材迭代链路的效果变化曲线
- 作为用户，我要能对比不同人群包在同一产品上的表现差异

**功能要点**：

- 产品维度的投放趋势图（消耗、CTR、CVR、CPM 随批次变化）
- 素材迭代链路可视化（v1 → v2 → v3 的指标变化折线图）
- 人群包对比表（同一产品下不同人群包的关键指标对比）

## 5. 非功能性需求


| 项目       | 要求                                     |
| -------- | -------------------------------------- |
| **性能**   | CSV解析 < 3s（100行以内）；复盘生成 < 60s（流式输出）    |
| **数据安全** | 投放数据仅存储在本地 PostgreSQL，不上传外部服务（LLM调用除外） |
| **兼容性**  | CSV编码支持 UTF-8 和 GBK（千川导出默认GBK）         |
| **可扩展**  | 预留字段支持未来接入千川API自动拉取数据                  |

## 6. 里程碑计划


| 阶段           | 内容                                | 优先级 |
| ------------ | --------------------------------- | --- |
| **P0 — MVP** | 批次管理 + CSV导入 + 人群包 + 复盘生成 + 知识库沉淀 | 必做  |
| **P1 — 增强**  | 视频分析关联 + 迭代追踪 + 可编辑日志             | 必做  |
| **P2 — 分析**  | 跨批次趋势分析 + 人群对比 + 素材迭代链路图          | 后续  |
| **P3 — 自动化** | 千川API对接 + 自动拉取数据 + 定时复盘           | 远期  |
